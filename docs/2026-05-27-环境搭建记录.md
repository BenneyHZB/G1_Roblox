# 环境搭建与开发流程

用 mise 管工具、Rojo 把 `src/` 同步到 Roblox Studio。

---

## 一、首次准备（只做一次）

```powershell
winget install jdx.mise          # 安装 mise-en-place
# 重开 PowerShell

cd C:\Roblox_PJ\G1_Roblox       # 换成你的实际路径
mise trust
mise install                     # 按 mise.toml 安装 Rojo
```

Studio 插件：**插件市场 → 搜索 Rojo → 安装**（与 CLI 7.6.1 对应即可）。

---

## 二、日常开发流程

```
Cursor 打开项目 → mise run serve → Studio Baseplate → Rojo Connect → Play
```

| 步骤 | 操作 |
|------|------|
| 1 | Cursor：**文件 → 打开文件夹** → 项目根目录 |
| 2 | 终端：`cd` 到根目录 → `mise run serve`（不关窗口） |
| 3 | Studio：New → Baseplate |
| 4 | 插件 → Rojo → **Connect** |
| 5 | **Play**，确认 Output 有 server/client 日志 |
| 6 | 在 Cursor 改 `src/`，保存即同步 |

---

## 三、项目里有什么

```
G1_Roblox/
├── mise.toml              # Rojo 版本 + 快捷命令
├── default.project.json   # 磁盘 ↔ Studio 映射
├── assets/
│   ├── map/               → Workspace.Map（关卡物体）
│   └── templates/         → ReplicatedStorage（敌人模板等）
└── src/
    ├── server/            → ServerScriptService.Server
    ├── client/            → StarterPlayerScripts.Client
    └── shared/            → ReplicatedStorage.Shared
```

**场景在哪：** 关卡物体也在仓库里（`assets/map`），Connect 后出现在 `Workspace.Map`；不必每次手动拖。

**复杂模型：** Studio 导出 `.rbxmx` → 放进 `assets/` → 改 `default.project.json` 的 `$path` → 保存即同步。

**两种打开 Studio 的方式：**

| 方式 | 用途 |
|------|------|
| Baseplate + Rojo Connect | 日常开发（推荐） |
| 打开 `build/place.rbxlx` | 快照/分享（需先 `mise run build`） |

---

## 四、换电脑 / 换路径

项目全是相对路径，文件夹可随意移动。

换路径后：

1. Cursor 重新打开新路径
2. 新路径下 `mise trust`（若提示未信任）
3. `mise install` → `mise run serve`
4. Studio 继续用 Baseplate + Connect

---

## 五、排错

| 现象 | 处理 |
|------|------|
| `mise` 不是内部或外部命令 | 重开终端；或 `$env:Path += ";$env:LOCALAPPDATA\Microsoft\WinGet\Links"` |
| Config files are not trusted | 在项目根目录执行 `mise trust` |
| no tasks defined | 不在项目根目录，先 `cd` 到含 `mise.toml` 的目录 |
| Connect 失败 | 确认 `mise run serve` 在跑；Rojo 插件已装 |
| 改了代码 Studio 没变化 | 确认已 Connect；保存文件；必要时断开再 Connect |
