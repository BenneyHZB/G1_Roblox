# Roblox 开发环境搭建记录

**日期：** 2026-05-27  
**项目：** roblox-starter  
**目标：** 用 mise 管理工具链，用 Rojo 把磁盘代码同步到 Roblox Studio，今天跑通最小可运行项目。

## 一、今天装了什么

| 工具 | 版本 / 说明 | 安装方式 |
|------|-------------|----------|
| **mise** (mise-en-place) | 2026.5.15 | `winget install jdx.mise` |
| **Rojo** | 7.6.1 | 写在项目 `mise.toml` 里，进项目目录后 `mise install` |
| **Roblox Studio** | 用户本机已有 | [create.roblox.com](https://create.roblox.com) |
| **Git** | 2.54.0 | 用户本机已有 |

**为什么用 mise 而不是 Aftman：** Aftman 已停止维护；mise 是通用开发环境管理器，可在 `mise.toml` 里固定 Rojo 版本，并定义 `serve` / `build` 等任务。

```
roblox-starter/
├── mise.toml                 # 固定 Rojo 版本 + 快捷任务
├── default.project.json      # Rojo：磁盘目录 ↔ Studio 服务树映射

    ├── client/
    │   └── init.client.luau  # 客户端入口
    └── shared/
```

### `default.project.json` 映射关系

| 磁盘路径 | Studio 位置 |
|----------|-------------|
| `src/server` | `ServerScriptService.Server` |
| `src/client` | `StarterPlayer.StarterPlayerScripts.Client` |
| `src/shared` | `ReplicatedStorage.Shared` |

### `mise.toml` 里的任务

| 命令 | 作用 |
|------|------|
| `mise run serve` | 启动 Rojo 同步服务（开发时保持运行） |
| `mise run build` | 导出 `build/place.rbxlx` |
| `mise run plugin` | 尝试把 Rojo 插件装进 Studio（本机需改用手动装插件） |

## 三、今天跑通的流程（日常开发）

1. **在 Cursor 中打开项目根目录**（必须是包含 `mise.toml` 的那一层）。

2. **终端里启动同步**（必须在项目目录内执行）：

   ```powershell
   cd <你的项目路径>\roblox-starter
   mise trust          # 换路径或首次克隆后可能需要
   mise install        # 确保 Rojo 已安装
   mise run serve      # 保持窗口不关
   ```

3. **Roblox Studio：**
   - 打开空 **Baseplate**，或继续用当前地方；
   - **插件 → Rojo → Connect**；
   - 点 **Play**，在 **Output** 中应看到 server/client 的 print。

4. **写代码：** 在 Cursor 里改 `src/**/*.luau`，保存后 Studio 在已 Connect 时会自动同步。

5. **验证同步：** 改 `src/server/init.server.luau` 里的 `print` 文案，保存后再 Play，Output 应显示新内容。

## 五、移动整个项目文件夹

**可以随便换位置**，项目内配置全部是**相对路径**，不依赖 `C:\Users\c\Projects\...`。

换位置后只需注意：

| 事项 | 说明 |
|------|------|
| 终端 | 在新路径下 `cd` 再执行 `mise run serve` |
| Cursor | **文件 → 打开文件夹**，选新路径 |
| `mise trust` | 若提示未信任配置，在新目录执行一次 `mise trust` |
| Studio | 若还开着旧的 `build/place.rbxlx`，用新路径重新打开或改用 Baseplate + Connect |
| Git | 移动文件夹不影响仓库；若已 `git init`，远程地址不变 |

**不需要改的文件：** `mise.toml`、`default.project.json`、`src/**`、`.gitignore`（均无绝对路径）。

**可选：** 全局工具（mise 本身）装在系统里，与项目路径无关。

## 六、两种打开 Studio 的方式（别混用）

| 方式 | 适合 |
|------|------|
| **`mise run serve` + Rojo Connect** | 日常开发：代码跟 Cursor 实时同步 |
| 打开 **`build/place.rbxlx`** | 查看/分享快照；改磁盘代码不会自动进这个文件，除非重新 `mise run build` |

长期建议：Studio 用 Baseplate（或模板）+ **Connect**，不要依赖每次从 `build/place.rbxlx` 打开。

## 七、已知问题与备忘

1. **`mise run serve` 必须在项目根目录执行**  
   在 `C:\Users\c` 等上级目录会报错：`no tasks defined`。

2. **`rojo plugin install` 失败**  
   报错与 Roblox 注册表/安装路径有关；已通过 Studio 插件市场手动安装 Rojo 插件即可。

3. **Git**  
   若当时未在项目里 `git init`，可在项目根自行执行；`.gitignore` 已忽略 `build/`、place 文件等。
