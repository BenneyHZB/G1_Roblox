# 项目指南

简要说明：**改什么功能去哪个文件**，以及发布流程。

---

## 日常开发

1. `mise run serve`（终端不关）
2. Studio → Rojo **Connect** → **Sync In**
3. 改代码保存 → Play 测试

Output 应出现 `[roblox-starter] Server started`。

---

## 目录与 Studio 对应

| 磁盘 | Studio |
|------|--------|
| `src/server/` | ServerScriptService.Server |
| `src/client/` | StarterPlayerScripts.Client |
| `src/shared/` | ReplicatedStorage.Shared |
| `assets/map/` | Workspace.Map |
| `assets/templates/Weapons/` | ReplicatedStorage.Weapons |
| `assets/templates/EnemyTemplate.rbxm` 等 | ReplicatedStorage 下同名 |

映射规则见根目录 `default.project.json`。

## 改功能 → 改哪里

### 数值与全局配置（优先看这里）

**文件：** `src/shared/GameConfig.luau`

| 想改的内容 | 配置项 |
|------------|--------|
| 开局金币、击杀金币 | `StartingGold`、`GoldPerWaveEnemyKill` |
| 奖牌起始值、跨服存档 | `StartingTrophies`、`TrophyDataStore` |
| 公主血量、受击伤害 | `PrincessMaxHp`、`PrincessDamagePerHit` |
| 波次间隔、每波怪数量 | `FirstWaveDelay`、`WaveIntermission*`、`WaveBaseEnemyCount` |
| 怪物随波次变强 | `WaveEnemyHealthPerWave`、`WaveEnemySpeedPerWave` 等 |
| 武器伤害、攻速、范围 | `WeaponStats`（键名 = Tool 名，如 `Ban Hammer`） |
| 武器店门槛与商品 | `WeaponShop.Items`、`ItemOrder` |
| 强化店价格与加成 | `ShopUpgrades` |
| 金币农场档位 | `GoldFarm.Tiers` |
| 训练木桩等级与奖励 | `Training` |
| 房间人数上限 | `MaxPlayers`（Studio 游戏设置里也设 4） |
| 地图头顶标签 | `MapLabels.Locations` |

### 波次 / 公主 / 重开

| 功能 | 文件 |
|------|------|
| 刷怪、公主扣血、波次循环 | `src/server/WaveDefense.luau` |
| 公主死亡后重开、清场 | 同上 `requestRestart`、`resetRoundState` |
| 重开清零（保留奖牌） | `src/server/PlayerReset.luau` |
| 左上角公主/波次 UI、重开按钮 | `src/client/Hud.luau` |
| 公主受击音效（全局） | `src/client/PrincessHitSound.client.luau` |

### 战斗与武器

| 功能 | 文件 |
|------|------|
| 攻击判定、伤害、击杀金币 | `src/server/PlayerCombat.luau` |
| 客户端挥砍动画/音效 | `src/client/Combat.client.luau` |
| 武器店领取逻辑 | `src/server/WeaponShop.luau` |
| 武器店 UI | `src/client/WeaponShop.client.luau` |
| Tool 查找、Handle 处理 | `src/shared/WeaponUtil.luau` |
| **武器模型文件** | `assets/templates/Weapons/*.rbxm` → Sync 后在 `ReplicatedStorage.Weapons` |

**加新武器：**  
1. Save to File → `assets/templates/Weapons/新名字.rbxm`（Tool 名与 `WeaponToolName` 一致或可模糊匹配，如 `BanHammer` ↔ `Ban Hammer`）  
2. 在 `GameConfig.WeaponShop.Items` + `WeaponStats` + `ItemOrder` 各加一项  
3. Sync In，Play 看 Output `[WeaponShop]` 是否 ✓  

### 经济 / 奖牌 / 强化

| 功能 | 文件 |

### 训练场 / 金币农场 / 指引

## 场景与模型放哪

| 内容 | 放哪 |
|------|------|
| 公主、武器店、刷怪点、农场区域 | `assets/map/`（或 Studio 改完 Save to File 回仓库） |

## 发布流程（简要）

1. **Sync In**，Studio 内 Play 测一遍（波次、武器店、奖牌存档）
2. **游戏设置**（主页 → 游戏设置）  
   - 最大玩家数 **4**  
   - 安全 → **启用 Studio 访问 API 服务**（本地测存档）  
   - 权限先 **私人/好友**
3. Studio：**文件 → 发布到 Roblox** → 选体验 → 发布
4. 用**发布后的链接**进游戏再测（奖牌跨服、四把武器等）
5. 之后每次改代码：**Sync In → 再发布**

可选：`mise run build` 生成 `build/place.rbxlx` 做快照。

**说明：** 奖牌走 DataStore，需**已发布**体验；金币/武器/强化仍是单服一局，公主死亡重开会清零（奖牌保留）。

## 排错速查

| 现象 | 检查 |
|------|------|
| 逻辑全没、UI 空 | Output 红字；Rojo 是否 Connect；`init.server` 是否报错 |
| 武器领不了 | Output `[WeaponShop]`；Tool 是否在 `ReplicatedStorage.Weapons`；名称是否对上 |
| 奖牌不跨服 | 是否已发布；Studio 是否开 API；Output `[PlayerTrophies] 载入` |
| Rojo 改了 Studio 没变 | Sync In；`default.project.json` 是否包含该路径 |
